\section{Pipeline validation}
\label{sec:validation}

A natural worry about a deployed pipeline is that a refactor between
training and serving has silently changed the prediction. We rule this
out by re-running the deployed pipeline on a date \emph{inside} the
training window (2022-12-30) and comparing element-wise to the
cluster's stored prediction.

\paragraph{Setup.}
The cluster's stored prediction for 2022-12-30 is bundled in
\texttt{pretrained\_models/baseline/dump/} and reports 6.54~\% overall
MAPE for that day (a single 24-h forecast). The reproduction script
\texttt{scripts/validation/reproduce\_dec30\_2022.py} fetches HRRR f00
analyses from AWS S3 via the deployed pipeline's
\texttt{space/hrrr\_fetch.py}, loads training-CSV demand for the
24-hour history and 24-hour truth windows, loads the deployed
checkpoint \texttt{space/checkpoints/best.pt}, and runs the same
normalize / forward / denormalize chain the live serving stack uses.

\paragraph{Result.}
The deployed pipeline produces $6.41~\%$ MAPE
[$95~\%$ CI: $4.99$--$7.69$] vs the cluster's $6.54~\%$
[$5.02$--$7.92$] on the same day; the paired bootstrap difference is
$-0.13$~pp [$-0.23$, $+0.00$]. Element-wise prediction differences are
bounded by 94~MW (3.6~\% of the cluster's value at the worst point).
Figure~\ref{fig:validation} overlays the two curves per zone.

\begin{figure}[h]
\centering
\includegraphics[width=0.95\linewidth]{validation_dec30_2022_overlay.png}
\caption{Pipeline-equivalence check on 2022-12-30. Solid = ground truth
demand. Dashed = cluster's stored prediction. Dotted = live pipeline
rerun. The two prediction curves visually overlap in every zone;
overall MAPE matches within 0.13~pp with a bootstrap CI that excludes
$+0.00$ on the upper side, indicating a small (sub-1~MW per cell)
systematic offset.}
\label{fig:validation}
\end{figure}

The 0.13-pp gap is not literally zero, indicating that some
non-determinism (kept-deterministic subsystems aside, the model has no
seeds; Appendix~\ref{app:hparams}) or floating-point ordering in the
re-implementation produces a sub-MW per-cell systematic. The size is
negligible relative to the deployment-period drift reported next, so we
treat the deployed pipeline as functionally equivalent to the
training-time evaluation. Any subsequent gap is attributable to
input-distribution change, not to a serving bug.
